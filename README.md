# 🦑Coqid-game (코키드 게임)

> **"당신의 플러그인은 살아남을 자격이 있습니까?"**  
> 설치 후 방치된 Codex 플러그인과 스킬의 활용도를 심사하고 정리하기 위한 **안전한 플러그인 생존 서바이벌 대시보드**입니다. 오징어게임 테마의 시각적 요소와 탈락 권고 메커니즘을 결합하여 자원을 정리할 수 있도록 돕습니다.

---

## 프로젝트 개요

Coqid-game은 설치된 Codex 플러그인의 사용량(주간/월간), 비용, 평가, 미사용 기간을 분석하여 **생존 점수(Survival Score)** 를 산출합니다.

* **정리 권고**: 점수가 낮아 탈락 위기에 처한 플러그인에 경고를 보냅니다.
* **사용성 검증**: 적절한 비용과 충분한 사용량을 보이는 플러그인은 'Safe' 등급으로 생존합니다.
* **안전 설계 (Review-only)**: 본 서비스는 권고 대시보드로, **실제 플러그인을 임의로 삭제하거나 훼손하지 않습니다.** 모든 정리는 개발자가 수동으로 처리합니다.

---

## 핵심 기능

1. **생존 심사 (Survival Check)**
   - 주간/월간 호출 수, 유지 비용, 방치 기간, 평가를 조합하여 `0~100점` 범위의 결정론적 생존 점수를 산출합니다.
2. **3단계 상태 및 테마 시각화**
   - **Safe (생존)**: 활발하게 사용되며 비용 효율이 높은 플러그인 (70점 이상).
   - **Reminder Recommended (달고나 미션)**: 재사용을 넛지하는 황금색 펄스 애니메이션 카드 (40~69점). 점수 구간별 4단계 넛지 메시지를 제공합니다.
   - **Deletion Recommended (탈락 권고)**: 비용 대비 활성도가 낮아 삭제를 검토하는 미세 플리커링 핑크 카드 (40점 미만).
3. **글로벌 아레나 리더보드 (Global Arena)**
   - 주간/월간 단위로 전체 사용자 관점에서의 상위 플러그인 랭킹을 시뮬레이션하여 비교합니다.
   - 1위 플러그인은 황금 테두리로 강조되며, 효율성과 기여도에 따라 6종의 서바이벌 뱃지가 부여됩니다.
4. **안전성 및 예외 처리 (Fail-safe)**
   - 데이터가 비어있거나(`Load Empty`) 유효하지 않은 형식(`Load Malformed`)이어도 중단 없이 정중한 빈 상태/오류 메시지를 출력합니다.
5. **접근성 지원 (Reduced Motion)**
   - `prefers-reduced-motion` 미디어를 지원하여 동적인 펄스 및 플리커 애니메이션을 부드러운 정적 화면으로 자동 전환합니다.

## 시작하기

### 1. 사전 요구사항
본 프로젝트는 별도의 외부 프레임워크나 라이브러리 없이 실행 가능한 웹 애플리케이션입니다. 빌드 및 개발 서버 구동을 위해 `Node.js` (v18 이상 권장) 환경이 필요합니다.

### 2. 빌드 실행
로컬 정적 자산을 생성하고 `dist/` 폴더에 복사합니다.
```bash
npm run build
```

### 3. 개발 서버 구동
로컬 테스트 및 브라우저 시연을 위한 초경량 개발 서버를 구동합니다.
```bash
npm run dev
```
서버 실행 후 웹 브라우저를 열고 `http://localhost:4173` (또는 `PORT` 환경 변수로 지정한 포트)에 접속하면 시각화 대시보드를 시연하고 다양한 모의 데이터를 테스트할 수 있습니다.

### 4. 자동화 테스트 수행
Scoring 엔진, 리더보드 랭킹 공식, 그리고 실제 삭제 방지 등의 안전 제약조건이 정상 작동하는지 검증합니다.
```bash
# 내장 테스트 러너 실행 (17개 핵심 비즈니스 로직 테스트)
npm test
```

### 5. 통합 스모크 테스트 실행
중요 화면 문구, 필수 버튼 및 상태 메시지 존재 여부를 사전에 정적으로 검사합니다.
```bash

npm run smoke
```

## 생존 심사 및 정규화 공식 가이드

### 1. 생존 점수 산출 공식 (`scoring.js`)

생존 점수는 다음과 같은 세부 지표들의 결정론적 합산으로 연산되며, 최종 점수는 최소 `0점`에서 최대 `100점` 사이로 보정(Clamp)됩니다.

$$\text{Survival Score} = 35 + \text{주간 사용 점수} + \text{월간 사용 점수} + \text{사용자 평가 점수} + \text{최근 사용 보너스} - \text{기본 비용 패널티} - \text{고비용 방치 패널티}$$

#### 세부 산출 기준

* **주간 사용 점수 (최대 25점)**: 주당 호출 횟수 $\times 3$
* **월간 사용 점수 (최대 20점)**: 월당 호출 횟수 $\times 1$
* **사용자 평가 점수 (최대 20점)**: 별점(1~5점) $\times 4$ (데이터가 누락되거나 없는 경우 기본값 `12점` 부여)
* **최근 사용 보너스 (가중치 변화)**:
  - 마지막 사용이 7일 이내인 경우: `+15점`
  - 마지막 사용이 8일 이상 30일 이내인 경우: `+8점`
  - 마지막 사용이 31일 이상 90일 이내인 경우: `0점`
  - 마지막 사용이 91일 이상인 경우 (오래 방치됨): `-10점`
* **기본 비용 패널티**:
  - `high` 등급 비용인 경우: `-18점`
  - `medium` 등급 비용인 경우: `-8점`
  - `low` 등급 비용인 경우: `-0점`
* **고비용 방치 패널티**:
  - 비용 수준이 `high`이면서 주간 사용 횟수 1회 이하, 그리고 월간 사용 횟수 5회 미만인 비효율 플러그인인 경우: 추가 감점 `-12점`

---

### 2. 리더보드 랭킹 스코어 공식 (`leaderboard.js`)

글로벌 아레나 리더보드는 단순 생존력뿐만 아니라 **실제 활용 가치가 높은 플러그인**이 최상위에 위치하도록 설계되었습니다.

$$\text{Leaderboard Score} = \text{사용 주기 가중치 (최대 60점)} + (\text{생존 점수} \times 0.4)$$

* **사용 주기 가중치 (최대 60점)**:
  - **주간 랭킹**: $\min(\text{주간 사용 횟수} \times 10, 60)$
  - **월간 랭킹**: $\min(\text{월간 사용 횟수} \times 3, 60)$

---

### 3. 서바이벌 뱃지 시스템

리더보드 진입 시 각 플러그인의 생존성과 기여도에 따라 다음과 같은 6가지 전용 서바이벌 뱃지가 실시간으로 매핑됩니다.

| 뱃지 명칭 | 출력 표기 | 매핑 조건 |
| :--- | :--- | :--- |
| **ELIMINATED** | Eliminated | 생존 점수가 40점 미만인 경우 (탈락 대상) |
| **SURVIVOR** | Survivor | 생존 점수가 85점 이상인 최우수 생존 플러그인 |
| **EFFICIENT SURVIVOR** | Efficient Survivor | 비용 부담이 `low`이며 월간 사용량이 10회 이상인 가성비 우수 플러그인 |
| **WEEKLY CHAMPION** | Weekly Champion | 주간 리더보드에서 주간 사용량이 8회 이상인 최고 활성 플러그인 |
| **MONTHLY CHAMPION** | Monthly Champion | 월간 리더보드에서 월간 사용량이 20회 이상인 최고 활성 플러그인 |
| **ACTIVE** | Active | 위 조건에 해당하지 않는 정상 동작 플러그인 |
